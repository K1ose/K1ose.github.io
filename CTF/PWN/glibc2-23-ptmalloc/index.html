__libc_malloc

跳过了初始化和定义的部分，先从__libc_malloc开始；

首先判断了是否定义了hook函数，如果hook != NULL，则return到该函数地址，如果没有hook函数，则执行：

指针指向一个空闲的main_arena，然后将该区域标志为已分配；

_int_malloc函数调用后获得内存地址，如果内存地址申请失败，则会再次尝试获取空闲的arena，并malloc一个堆块；

成功malloc后，将解锁分配的区域；

如果申请内存失败，申请到的是mmap的内存，申请到的内存在分配的arena中；

返回内存；

关于__glibc_unlikely和__builtin_except()的内容，参考一篇博客；

这里再次判断是否没有申请到内存，如果没有申请到，将会调用sysmalloc尝试获取内存地址，如果获取到了，则调用alloc_perturb清理内存空间，返回内存地址；

如果申请到了内存空间，判断申请的内存大小是否小于get_max_fast，即fastbin中有满足size的堆块，则根据大小获取索引idx。定义fastbin指针指向当前索引的fastbin链表；

接着将内存地址指向该fastbin链表，并且fastbin = fastbin -> fd；

如果申请到了内存空间，判断 victim_size对应的fastbin索引是否不等于size对应的fastbin索引，即是不是符合这个链表的大小，如果不符合则抛出异常，符合则检查chunk的合法性。随后将victim chunk指针转换为用户数据区指针，调用alloc_perturb()对用户数据区进行初始化；

如果是在smallbin的范围内，则获取对应的索引，定位索引所在的区域。接着让victim = last(bin)，即为smallbin->bk，判断是否为smallbin，即判断smallbin是否为空。接着判断smallbin->bk是否为0，是的话就进行初始化；

如果victim -> bk -> fd != victim，则抛出异常。设置下一个chunk的PREV_INUSE为1，将victim从链表中取下，即smallbin -> bk = smallbin -> bk -> bk，smallbin -> bk -> fd = smallbin。然后判断分配的区域是否为main_arena，如果不是则设置victim->size为NON_MAIN_ARENA。接着检查合法性，将victim chunk指针转换为用户数据区指针，调用alloc_perturb()对用户数据区进行初始化，并返回该地址指针；

如果属于largebin的大小范围，则获取largebin的索引，调用have_fastchunks查看当前分配区域内是否存在fastbin，如果存在则进行初始化；

如果unsortedbin不为空，则令bck = victim->bk = unsortedbin->bk，判断size的合法性，如果不合法，则抛出异常；

如果大小属于smallbin，而且只存在一个unsortedbin，并且当前的unsorted chunk属于last_remainder，unsortedbin的size还大于申请的nb+MINSIZE，则切分这个unsorted chunk，剩下的部分放回链表；

如果剩下的chunk大小不属于smallbin范围，则将剩下的chunk的fd_nextsize和bk_nextsize置NULL；

设置victim的prer_inuse和main_arena标志位，设置last_remainder的prev_inuse标志位和last_remainder下一个块的prev_inuse标志位，将victim转换为用户数据区地址，调用alloc_perturb()对用户数据区进行初始化；

从unsorted bin中移除已经分配的victim chunk，unsortedbin->bk = victim->bk，victim->bk = unsortedbin；

判断当前切割出来的unsortedbin_size是否等于申请的chunk size，如果是，则设置victim的下一个chunk的prev_inuse位以及main_arena位，检查chunk的合法性，将victim转换为用户数据区地址，调用alloc_perturb()对用户数据区进行初始化；

如果切割出来的chunk属于smallbin的大小范围，则获得其前后堆块的指针；

否则获得其前后堆块的指针；

如果largebin不为空，则设置victim的prev_inuse为1；

将victim按大小顺序插入到largebin中；

设置分配区的binmap，将victim插入目标位置；

迭代器+1，当迭代器数值大于10000时break；

如果申请的size不在smallbin范围内，则定位到largebin；

如果largebin不为空，并且victim chunk size大于申请的size，则victim等于下一个更小的chunk，直到大小正合适位置；

如果victime不是最后一个largebin，并且size等于前一个chunk的size，则让victim为前一个chunk，即大小相等的时候，挑后一个，这样避免了重新更新列表；

剩下的块减去申请的大小，然后调用unlink脱链victim chunk；

判断剩下的大小是否小于MINSIZE最小可分配大小，即是否耗尽了这个chunk，如果是，则即将victim chunk的下一个chunk的prev_inuse设置为1，并设置main_arena标志位；

如果没有耗尽，则获取剩下的堆块，放到unsortedbin中；

如果bck->fd->bk != bck，则抛出异常，否则将剩余堆块放入unsortedbin中；

如果剩余堆块不属于smallbin范围，则将其fd_nextsize和bk_nextsize置为NULL，并设置victim的prer_inuse和main_arena标志位，设置last_remainder的prev_inuse标志位和last_remainder下一个块的prev_inuse标志位，将victim转换为用户数据区地址，调用alloc_perturb()对用户数据区进行初始化；

largebin索引+1，遍历largebin寻找满足要求的空闲块(使用binmap和block缩短检索时间)，如果找到则按照上诉方式分配victim；

victim指向top_chunk，size为victim的size，从top_chunk中分配chunk， 判断top_chunk_size是否大于等于(size+MINSIZE)， 如果是则切割top_chunk赋值给victim,将victim转换为用户数据区地址，调用alloc_perturb()对用户数据区进行初始化，返回地址指针；

如果存在fastbin，则初始化分配区域，如果申请的大小在smallbin范围内，则定位到smallbin，否则定位到larginbin；

如果不存在fastbin，则使用sysmalloc分配空间，使用alloc_perturb 清空内存，返回地址指针；

malloc_consolidate

malloc_consolidate是对堆的一种整理，从fast bin中删除每个chunk并将其合并，然后将其放入unsorted bin中；

可以看到它对fastbin的操作；

一些成员变量和定义；

如果max_fast不为0，则清空分配区have_fastbin标志，否则执行malloc_init_state进行初始化；

获得unsorted bin，fastbin[0]，fastbin[MAX]的地址，遍历fastbin中的所有chunk，对每个chunk做如下操作；

获得chunk_size,下一个块的地址与next_chunk_size；

如果上一个堆块是空闲的，则合并chunk，并使用unlink将上一个块从bin中释放；

如果下一个堆块不是top_chunk，则判断下一个chunk是否空闲，如果是则合并chunk，并使用unlink将下一个块从bin中释放；

将chunk插入unsortedbin头部，返回地址指针；

将chunk与top_chunk合并；

如果max_fast为0，则初始化堆块状态并检查；

free

__libc_free

同样是从__libc_free开始，代入参数为要进行free操作的指针；

查看有无钩子函数，如果有则进行跳转执行；

如果内存地址指针为0，则直接返回，

将指针转换为用户内存chunk指针，并判断chunk是否是由mmap分配得到的，如果是，则判断：

  1. 是否没有禁用动态阈值
  2. chunk大小是否大于当前mmap阈值的大小
  3. chunk大小是否小于等于mmap默认阈值的最大值

如果满足条件，则调整当前阈值为chunk的大小，并且mmap的紧缩阈值为该chunk大小的两倍。随后释放该mmap分配的chunk并返回；

如果chunk不是mmap分配的chunk，则获得chunk分配的区域arena，调用_int_free；

_int_free

brk申请的堆块具体是由_int_free实现释放的，传入参数为分配区指针av，待free的chunk指针p以及锁标志have_lock；

判断指针p合法性以及是否对齐，如果不满足则抛出指针异常；

如果chunk的大小小于分配的最小单位，并且也没有对齐，则抛出大小异常；

检查chunk的状态是否合法；

如果chunk_size小于等于fast bin的最大值，则放到fast bin中；

如果chunk的下一个chunk_size小于等于2*SIZE_SZ，或者下一个chunk的chunk_size大于等于arena的system_mem；

分配区上锁，判断分配区是否加锁或者next_chunk_size是否小于等于2*sizeof(INTERNAL_SIZE_T)或者next_chunk_size是否大于等于av->system_mem，如果满足，则抛出下一个chunk_size异常；

解锁；

设置分配区av的标志位flag的FASTCHUNKS_BIT为1，根据size寻找到fastbin的索引idx，然后根据分配区和索引获得对应的&fastbin指针；

定义两个个chunk指针，old指向&fastbin；

如果判断指针所指向的chunk是否为要释放的chunk，这里很明显的漏洞就是fastbin double free，如果是则抛出double free异常；

如果分配区上锁，且&fastbin不为空，则获取索引，并将chunk插入到fastbin头部；

如果分配区上锁，并且fastbin不为空，而且索引不相等，则抛出fastbin插入的异常；

如果chunk不是mmap分配的，分配区没有上锁，则上锁；

获取下一个chunk的指针，如果chunk是top_chunk，则抛出异常；

如果存在连续分配区并且next_chunk指针大于等于top_chunk_addr + top_chunk_size的地址，则抛出越界异常；

如果next_chunk的标志位prev_inuse为0，表示chunk已经被标记为free状态，则抛出异常；

如果next_chunk_size<=2*SIZE_SZ或者next_chunk_size>=av->system_mem，抛出异常；

判断prev_chunk是否是free状态，是的话，则合并并且unlink；

如果next_chunk不是top_chunk则获取next_chunk_inuse；

如果next_chunk为空闲状态，则unlink后合并，否则设置其prev_inuse为0；

获取分配区的unsorted_chunks链表指针，判断 unsorted_bin->fd->bk是否不等于unsorted_bin，否则抛出unsorted bin异常；

将chunk插入到unsorted bin中，然后判断其大小是否属于small bin范围，如果不是则清空chunk的fd_nextsize和bk_nextsize区域，设置chunk的pre_inuse和size，最后检查chunk的free状态是否正常；

如果其next_chunk已经是top_chunk，则将当前chunk合并到top_chunk中，设置prev_inuse，检查top_chunk的状态；

如果被释放的堆块的chunk_size>=65536，即fastbin合并的阈值，则触发malloc_consolidate合并分配区中的fastbin；

判断分配区是否为主线程区域，如果是则判断是否top_chunk_size>=mp_.trim)threshold收缩阈值，如果是的话，则系统调用搜索，将多余内存归还操作系统；

如果是非主线程区域，尝试找到该区域的top_chunk，并收缩top_chunk；

分配区解锁，执行munmap_chunk释放chunk；
